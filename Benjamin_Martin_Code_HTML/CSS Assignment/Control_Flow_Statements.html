Control Flow Statements

By Benjamin Martin

What are Control Flow Statements?

Control Flow Statements are statements used in programming to dictate the next set of operations the program will perform either with a binary decision or iterating through an operation until a specific condition is met.

All programs, whether basic or complex, require decision-making in some form or another. Without control flow statements, activities such as looping through arrays would be impossible. Programs would also lose their ability to perform more than one activity and would require the user to take the output and funnel it into the next program depending on what it is instead of letting the program do it for them.

Decision-Making

Statements used to determine if a set of operations will occur and if so under what conditions will need to be met in order to execute.

If-else, if-else if-else, and switch all perform the same, but should be selected based on how many decisions can be made.

If-else, if-else if-else will both execute "else" if no other condition is met. For switch, the default case will happen if no other case is found.

To leave out the else or default will mean no code is executed if no condition is met.

Looping Statements

Statements that involve iterating through a set of operations until a specific condition is met.

While: Loop that will repeat a set of operations until the condition to perform them is false. Will not execute at all if the condition starts false.

Do While: Loop that performs like a While look except that it will execute once before checking the condition's status.

For: Loop that executes a specific number of times. Usually dictated with an initialized variable that counts the number of repeats. When the counter makes the condition false, the loop will stop.
Loops can also contain loops (nested loops).

Branching Statements

Statements used to alter what happens in a specific iteration of a control flow statement.

Continue: Skips the current iteration and moves onto the next iteration.

Break: Skips the current loop and continues to the next line of code.

Return: End the current loop and returns to whichever method or function called it from.

Continue and break are used primarly in loops while return sees most utility in functions/methods that want to stop at diffrent parts to return some modified value.
