Maps

What are Maps?

A map is an object in Java that assigns keys to values. A map cannot contain duplicate values and each key must map to at most one value.

One of the most important things to understand about Maps is that they are NOT A COLLECTION. Maps do not inherit from the Collections interface.
Instead, maps use two interfaces: Map and SortedMap, and three classes: HashMap, TreeMap, and LinkedHashMap.

Why/When would you use a Map?

Maps would be used when you need to look up elements by specific keys. Some examples would be:

  • A map of error codes and their descriptors.
  • A map of zip codes and cities.
  • A map of managers and employees, in which a manager (key) is associated with the different employees (value) he/she manages.
  • A map of classes and students, operating in a similar fashion to the above example.


Methods of Map

  1. Object put(Object key, Object value): Inserts an entry.
  2. void putAll(Map map): Inserts a map into another map.
  3. Object remove(Object key): Deletes an entry containing a specified key.
  4. Object get(Object key): Returns the value at the specified key.
  5. boolean containsKey(Object key): Searches for the specified key.
  6. Set keySet(): Returns the Set view containing all keys.
  7. Set entrySet(): Returns the Set view containing all keys and values.

HashMap

HashMap provides the basic implementation of a map in Java. It's called HashMap because it uses a technique called Hashing,
which involves converting a large String into a smaller String that represents the same String, which helps with indexing and faster searches.
It uses a linked list internally to store its key-value pairs.

A few important features:
  • HashMap is part of java.util.
  • HashMap implements the Cloneable and Serializable interfaces.
  • HashMap does not allow duplicate keys but does allow duplicate values.
  • HashMap allows only one null key, but multiple null values.

TreeMap

TreeMap is sorted according to the natural ordering of its keys, or by a Comparator provided at map creation.
This provides a more efficient way of sorting and storing key-value pairs.
A few important features:
  • A member of the Java Collections Framework.
  • Does not allow null keys and throws a NullPointerException when one is attempted to be assigned. Null values can be associated with different keys, however.

LinkedHashMap

LinkedHashMap is identical to HashMap with the added feature of maintaining order of elements inserted.
A few important features:
  • Implements Map and extends HashMap.
  • Contains only unique elements.
  • May have only one null key and multiple null values.
  • Maintains insertion order.
