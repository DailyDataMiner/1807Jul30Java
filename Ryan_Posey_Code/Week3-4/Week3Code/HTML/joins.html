Joins in SQL

A SQL join statement is used to combine data or rows from two or more tables based on common fields between them.
To recognize how the result set of a specific join is processed, it's better to visualize each join in a venn diagram,
which I have provided for each important join. In addition, to demonstrate the result sets, let's use these two tables:

There are four main types of joins:
  • Inner Join
    • Selects all rows from both tables where values intersect. Only rows that are common in both tables will be returned. 

      •                                         
                                            select first_name, last_name, order_date, order_amount
                                            from customers c
                                            left join orders o
                                            on c.customer_id = o.customer_id
      first_name last_name order_date order_amount
      George Washington 07/4/1776 $234.56
      John Adams 05/23/1784 $124.00
      Thomas Jefferson 03/14/1760 $78.50
      Thomas Jefferson 09/03/1790 $65.50
  • Left Join
    • Selects all rows from the left table and any matching rows from the right table. 
      •                                         
                                            select first_name, last_name, order_date, order_amount
                                            from customers c
                                            left join orders o
                                            on c.customer_id = o.customer_id
                                            where order_date is NULL
      first_name last_name order_date order_amount
      George Washington 07/04/1776 $234.56
      John Adams 05/23/1784 $124.00
      Thomas Jefferson 03/14/1760 $78.50
      Thomas Jefferson 09/03/1790 $65.50
      James Madison NULL NULL
      James Monroe NULL NULL
  • Right Join
    • A mirror of left join. Returns all rows from the right table and any matching rows from the left. 
      •                                         
                                                select first_name, last_name, order_date, order_amount
                                                from customers c
                                                right join orders o
                                                on c.customer_id = o.customer_id
      first_name last_name order_date order_amount
      George Washington 07/04/1776 $234.56
      Thomas Jefferson 03/14/1760 $78.50
      John Adams 05/23/1784 $124.00
      Thomas Jefferson 09/03/1790 $65.50
      NULL NULL 07/21/1795 $25.50
      NULL NULL 11/27/1787 $14.40
  • Full/Outer Join
    • Selects all records from both tables. Duplicate values will be included. 
      •                                            
                                            select first_name, last_name, order_date, order_amount
                                            from customers c
                                            full join orders o
                                            on c.customer_id = o.customer_id
