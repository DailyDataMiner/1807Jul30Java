JAVA ACCESS MODIFIERS

Keywords that set the accessibility of variables, methods, and classes are known as Access Modifiers. In the Java language, there exists four different types of modifiers: Public, Protected, Default, and Private. A top level class can only use Public and Default while the member level case use any of the four. Access modifiers, at least their concept, we're seen as early as 1972 in the Simula language in the form of the protected and hidden declarations. Even back in the early days of programming, encapsulation and the ability to regulate visibility were crucial in regards to security, maintainability, and flexibility.

Access Modifiers in Variables

Member variables inside a class can be declared with or without (default) an access modifier. Unlike classes, they get access to two more modifiers: private and protected. Access modifiers are written out first when a variable is being declared, followed by the non-access modifier, datatype, then variable name.
Public variables can be accessed anywhere in the program, and thus considered bad for encapsulation. Only use public when you want a member variable to be easily accessed, such as when you want to use a constant. In terms of encapsulation, private is the best modifier since it prohibits access to member variables from outside classes and needs getters and setters to interact with it. One caveat about private variables is that while they are "inherited" by subclasses, they cannot access them. Protected and default access modifiers do allow inheritance and access from the subclass, but are less restrictive with protected allowing the local package and subclasses outside the package access and default only allowing classes in the local package access.

Access Modifiers in Methods

Methods are also declared with access modifiers. They can have public, protected, default, or private access modifiers.
Like variables, public methods are accessible from anywhere in the program, while default and protected methods are only available inside the local package (default) as well as sub-classes in outside package (protected). Private methods can only be called within the class and are not accessible from a subclass, unless its through another inherited method.
When overriding an inherited method, the overriden method must have a declared access modifier less restrictive than the original method. This is because the child class is an instance of the parent and must present at least the same usable methods. Abstract methods cannot be private since they need to be inherited to be implemented.
Since constructors are methods, they too are declared with access modifiers. Normally, they are declared public, but can be declared private to limit instantiation. This is a core concept in the singleton design pattern.

Access Modifiers in Classes

Classes can only be declared public or default. Public classes can be accessed anywhere in the program while default classes can only be accessed in the local package. Even if the fields and methods are public, an outside package class cannot access those fields and methods in a default class.

Access Modifiers in Interface

Java interfaces can only be declared default or public. Fields in an interface are implicitly public static and cannot be declared private, protected, or default(package). Methods in an interface also follow these rules and the default keyword is replaced with Default Methods which do not have anything to do with access modifiers.

PUBLIC

  • Has the widest scope among all access modifiers.
  • Variables and methods can be accessed anywhere.
  • Outside packages still need to import the public class.
  • All public methods and variables from a class are inherited by its subclasses
  • Methods declared public in the superclass must be declared public in the subclass
  • Used rarely for member variables since it violates encapsulation

Protected

  • Variables, methods, and constructors can only be accessed from any class within the package or a subclass in another package.
  • Cannot be used on class and interface.
  • Cannot be used on methods and fields in an interface.
  • Methods declared protected in a superclass must be declared protected or public.
  • Protected is less restrictive than default.

Default

  • Default access modifiers are not explicitly declared for a class, field, or method.
  • Accessible to any other class in the same package.
  • Also known as "package".
  • Methods and fields in an interface are public by default.
  • Default is more restrictive than protected.

Private

  • Methods, variables, and constructors that are private can only be accessed within the class they are in.
  • Class and interface cannot be declared private.
  • Private variables can only be accessed from outside a class via public getter and setter methods inside the class.
  • The private modifier is the main way encapsulation is enforced since it hides data from the outside world.
  • Private methods and fields are not inherited.
