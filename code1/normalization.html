Database Normalization

This is the process of reducing data redundancy within a database. Reducing data redundancy is important because the cost of storing extra data is more expensive and can slow down your database performance. Normalization aims to eliminate Transitive dependencies. A transitive dependency is a functional dependency which holds by virtue of transitivity. A transitive dependency can occur only in a relation that has three or more attributes.

1st normal form

1NF

  • Eliminate repeating groups in individual tables
  • Create a separate table for each set of related data
  • Identify each set of related data with a primary key

2nd normal form

2NF

A table is in 2nd Normal Form if:

  • The table is in 1st normal form
  • All the non-key columns are dependent on the tableâ€™s primary key.

3rd normal form

3NF

A table is in third normal form if:

  • A table is in 2nd normal form.
  • It contains only columns that are non-transitively dependent on the primary key

Why do we normalize Databases?

Normalized Databases

Normaliztion is used for OLTP databases due to the fast write performance of a normalized db. It is easy to insert data on a bunch of small tables with only a few columns as opposed to writing to all the columuns on one table.

Denormalized Database

Denormalization is used for OLAP databases due to the fast read performance of a denormalized db. It is easy to read data from one big table as opposed to joining a bunch of small tables.
